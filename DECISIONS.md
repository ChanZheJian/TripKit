I built a simple website that gathers users' trip requirements and uses that information to call an AI API, which generates a travel itinerary and displays the results on the frontend.

Due to time constraints, I started with the core functionality first by supporting only basic inputs such as destination and trip duration. Once that was working, I added error handling to ensure the system would not generate unreliable or nonsensical results when the provided information was insufficient.

I progressively integrated additional features such as respect reality, adding weather api, estimated costs, and packing recommendations.
